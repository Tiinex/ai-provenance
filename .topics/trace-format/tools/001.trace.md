# Trace Format Tooling Tasks

This artifact now acts as the root task for the tooling branch.

The aim is to keep the remaining work concrete: preserve committed parent
links, browseable origins, and footer integrity without expecting a human to
reconstruct the whole chain by hand.

## Objective

Define and land the first bounded tooling slices needed to keep schema-lineage
maintenance practical.

## Done Criteria

- the tools branch is expressed as concrete child tasks rather than only as a
  design thread
- the first task split remains narrow enough to execute independently
- the branch still points back to the broader trace-format root without adding
  a second competing source of truth

## Scope And Constraints

- keep the first tooling tasks bounded and human-readable
- prefer reuse of existing TRACEABLE lineage helpers where that actually fits
- do not turn the tools branch into broad autonomous rewrite tooling too early

## Subtasks

- [001-1-markdown-preview-repo-resolver.trace.md](001-1-markdown-preview-repo-resolver.trace.md): define the resolver task for opening repo-matched GitHub links locally in markdown preview
- [001-2-schema-lineage-repair.trace.md](001-2-schema-lineage-repair.trace.md): define the repair task for bounded schema-lineage fixup behavior

## Risks

- tooling work can stay too abstract if the branch remains phrased only as
  direction instead of executable tasks
- shared lineage logic can drift if each task invents its own matching or
  repair rules
- repair and convenience work can blur together unless each child task keeps a
  narrow purpose
