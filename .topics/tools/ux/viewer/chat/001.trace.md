# Chat Projection Task

This task shapes the viewer's chat mode.

## Objective

Render trace state as a conversational projection that keeps the operator oriented without hiding provenance.

## Done Criteria

- ancestor entries remain readable as separators or threaded messages
- the current trace snapshot remains the same as the detailed view
- broken-lineage warnings and repair affordances remain visible
- labels stay short enough for the narrow chat pane
- the mode does not invent missing context or overwrite provenance with narration

## Current Read

The current implementation already has a `Chat` / `Detailed` toggle, chat separators, lineage banners, and repair buttons. This task should refine that existing projection rather than redesigning the panel from scratch.

## Scope And Constraints

The chat projection should:

- show the trace flow clearly
- keep ancestor and current entries distinguishable
- use concise headings and message blocks
- keep the repair entry point discoverable
- avoid turning the chat mode into a second editor

## Subtasks

- keep ancestor trace messages legible in sequence
- keep the broken-lineage banner short and actionable
- ensure the chat toggle still preserves the same snapshot state
