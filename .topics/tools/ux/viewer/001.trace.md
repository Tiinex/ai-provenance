# Trace Viewer UX Root

This topic tracks the viewer surfaces that render trace state for operators.

## Current Read

The current panel already switches between `Chat` and `Detailed`. That tells us the viewer is not one screen; it is at least two projections of the same snapshot:

- chat mode for a conversational, separator-driven trace read
- detailed mode for a denser inspection surface

The viewer should stay truthful to the same underlying state in both modes.

## Design Direction

The viewer tree should prefer:

- one snapshot, two projections
- chat mode when the goal is to read lineage as a narrative trace
- detailed mode when the goal is to inspect structure, provenance, and repair state
- repair affordances that remain reachable without turning the viewer into an editor
- concise labels that stay readable in a narrow panel

## Current Branches

- chat: the conversational projection of trace evidence and lineage
- detailed: the denser inspection mode for metadata, findings, and controls

## Next Artifacts

- define the chat projection task
- define the detailed inspector task
- revisit whether the viewer needs a future audit submode or whether detailed mode is enough for V1
