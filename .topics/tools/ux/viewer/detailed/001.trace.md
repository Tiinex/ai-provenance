# Detailed Viewer Task

This task shapes the detailed mode of the viewer surface.

## Objective

Render trace state as a structured inspector view that exposes provenance, findings, and repair state without relying on chat-like projection.

## Done Criteria

- detailed mode shows the same snapshot as chat mode
- structural metadata is easy to scan
- provenance and integrity remain explicit
- repair state is visible without reading chat prose
- the view stays faithful to the trace instead of summarizing away the key fields

## Current Read

The current panel already toggles into a non-chat detailed surface when chat mode is off. That surface should become the structured default for inspection work, not a generic dump.

## Scope And Constraints

The detailed view should:

- surface the envelope and current metadata cleanly
- show provenance and integrity in a stable order
- make validator findings easy to read
- keep repair actions visible when appropriate
- avoid turning every field into prose or narrative text

## Subtasks

- define the section order for the detailed inspector
- keep provenance and integrity lines readable in the narrow panel
- align the detailed view with validator and repair affordances
